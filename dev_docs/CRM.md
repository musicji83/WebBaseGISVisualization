## 1. 고객 프로필

### 1.1 주요 사용자 페르소나

#### 페르소나 A: GIS 초보 학습자
| 항목 | 내용 |
|------|------|
| **역할** | 프로그래밍 입문자, GIS 학습 중 |
| **목표** | QGIS와 웹 3D 시각화를 체험하며 GIS 기초를 배움 |
| **기술 수준** | Python 기초, QGIS 처음 사용 |
| **핵심 니즈** | 쉬운 실행 방법, 단계별 안내, 시각적 결과물 |
| **불만 요소** | QGIS 수동 조작이 복잡, 3D 뷰 크래시 |

#### 페르소나 B: 도시 계획 분석가
| 항목 | 내용 |
|------|------|
| **역할** | 건물 밀도/높이 분석, 도시 계획 보조 |
| **목표** | 특정 지역의 건물 분포를 빠르게 3D로 파악 |
| **기술 수준** | QGIS 중급, Python 기본 사용 가능 |
| **핵심 니즈** | 정확한 높이 데이터, 인터랙티브 탐색, 데이터 내보내기 |
| **불만 요소** | 데이터 수집~시각화 파이프라인이 수동 |

#### 페르소나 C: 데이터 시각화 개발자
| 항목 | 내용 |
|------|------|
| **역할** | 웹 기반 지리 데이터 대시보드 개발 |
| **목표** | GeoJSON → 3D 웹 시각화 레퍼런스 코드 확보 |
| **기술 수준** | JavaScript/Python 상급, deck.gl 경험 있음 |
| **핵심 니즈** | 코드 재사용성, 커스터마이징 용이성, 아키텍처 이해 |
| **불만 요소** | 문서 부족, 하드코딩된 좌표값 |

## 3. 고객 피드백 이력

### 3.1 이슈 및 대응

| 날짜 | 피드백 | 원인 | 대응 | 결과 |
|------|--------|------|------|------|
| 세션 1 | "반복 실행 시 QGIS 크래시" | QTimer 재진입(reentrancy) | `_processing` 플래그 추가 | ✅ 해결 |
| 세션 1 | "플러그인 수정이 반영 안 됨" | 심볼릭 링크가 아닌 복사본 | 플러그인 디렉토리에 직접 복사 | ✅ 해결 |
| 세션 1 | "Step 4 이후 항상 크래시" | setRenderer3D() 상태 오염 | Step 순서 변경 (5→4) | ✅ 해결 |
| 세션 2 | "3D가 아니라 2D 화면만 보임" | 3D Map View 미생성 | 웹 기반 3D 뷰어 구축 | ✅ 해결 |
| 세션 2 | "화면이 돌아가지 않음" | QGIS 2D 캔버스는 회전 불가 | deck.gl 인터랙티브 3D 뷰어 | ✅ 해결 |

### 3.2 만족도 평가

| 기능 | 기대치 | 실제 결과 | 만족도 |
|------|--------|----------|--------|
| 자동화 파이프라인 | 한 줄 실행 | `python3 run_all_steps.py` | ★★★★★ |
| 3D 시각화 | QGIS 3D 뷰 | 웹 기반 deck.gl (대체) | ★★★★☆ |
| 데이터 정확도 | OSM 건물 513개 | 513개 모두 로드 | ★★★★★ |
| 실행 안정성 | 크래시 없음 | Step 순서 조정으로 해결 | ★★★★☆ |

## 4. 향후 고객 요구 예측

### 4.1 단기 (v1.x)
- 좌표/반경을 파라미터로 받아 다른 지역도 분석할 수 있게 범용화
- 건물 이름/높이 기반 검색 및 하이라이트
- 범례 클릭으로 특정 높이 범위 필터링

### 4.2 중기 (v2.x)
- 여러 지역 비교 분석 (탭 형태)
- 건물 데이터 통계 대시보드 (평균 높이, 분포 차트)
- 시간대별 그림자 시뮬레이션 (태양 위치 기반)

### 4.3 장기 (v3.x)
- 실시간 OSM 데이터 업데이트
- 사용자 커뮤니티 공유 (특정 지역 3D 뷰 URL 공유)
- AR/VR 디바이스 지원
